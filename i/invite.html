    What is this?

    You're probably wondering what this page is all about. This is your personalized invitation page! Every invitation was sent with a unique QR code that allows us to communicate with you! As we receive your RSVP or obtain other wedding details, we'll update this page so check back often!

    Why was this page created?

    Our ceremony is going to be held in a garden that is open to the public. Our ushers have no way of identifying our guests from random spectators and since seating is extremely limited, we needed a way for our ushers to identify legitimate guests (also it was really fun for Jason to write the code behind it). Hence, ushers will be scanning your invitation and seeing your lovely picture when you come to the ceremony.

    But I don't like my picture!

    It's the picture we were able to easily find for you. If you don't like it, contact Jason or Ann and send us the picture you'd like ushers to see!

    The (fill in the blank) information is wrong

    We tried to take care to accurately capture the number of guests invited, the spelling of your name, and any food restrictions we're aware of. If we got it wrong, we're sorry - contact Jason or Ann with the right information and it'll be fixed!

    Why is the page blue / pink?

    The color will help our ushers quickly identify if you're on the groom's side (blue) or the bride's side (pink).

    Is it secure?

    Jason thinks so. The page itself is static HTML that makes an Asynchronous JavaScript And XML (AJAX) call to retrieve an encrypted JSON object. The specific encrypted JSON object that belongs to you is identified by a random 128-bit ID so it's infeasible for anyone to realistically guess and retrieve (a.k.a. "force browse") to your encrypted JSON object. This strategy is similar to the one used by Facebook and Google to protect photos - their file names are randomly generated in such a way that even though anyone could theoretically download it, it's infeasible as the file name itself is almost impossible to guess. This page takes things further as even if someone does guess the JSON object ID, the object itself is encrypted with a randomly generated 128-bit AES key that is unique per guest. The encrypted JSON object contains the guest profile information that you see above. The QR code on your invitation is unique to you and contains the ID and key that allows this static HTML page to read the contents. The ID and key are supplied as parameters in the anchor tag so it never traverses the network. So in short, it's reasonably secure.
